# optigo

Проект по изучению инструментов профилирования Go (`pprof`, `trace`) и итеративной оптимизации HTTP-сервиса.

## Стек технологий
* **Language:** Go 1.25+
* **Framework:** [Echo v4](https://github.com/labstack/echo)
* **Profiling:** net/http/pprof
* **Load Testing:** [hey](https://github.com/rakyll/hey)

---

## Проект: Нагрузочное тестирование и оптимизация

### Цель
Создать масштабируемый сервис, выявить узкие места (CPU/RAM) с помощью профилировщика и оптимизировать их, используя бенчмарки для подтверждения результата.

---

## Этап 1: Baseline

На данном этапе реализована "наивная" версия API со встроенными намеренными неэффективностями:
1. **Динамическая компиляция регулярных выражений:** `regexp.MustCompile` вызывается на каждый входящий запрос.
2. **Лишние аллокации:** Использование `fmt.Sprintf` в циклах для конкатенации строк.
3. **Обход JSON:** Стандартный `c.Bind` и создание новых слайсов без предварительного выделения памяти (`make`).

### Окружение для тестов
* **Инструмент:** `hey`
* **Конфигурация:** 20,000 запросов, 50 одновременных соединений.
* **Payload:** JSON-файл `data.json` (1000 элементов в массиве).

## Этап 2: Global Regexp & Memory Pre-allocation

### Изменения:
- Вынос `regexp.MustCompile` в глобальную переменную (компиляция один раз при старте).
- Предварительное выделение памяти под слайс: `make([]string, 0, len(req.Names))`.
- Замена `fmt.Sprintf` на конкатенацию строк.

**Вывод:** Производительность выросла, но профилирование `top20` показало, что `regexp` всё еще занимает ~6% CPU из-за внутренних блокировок.

## Этап 3: No Regexp (Byte Comparison)

### Изменения:
- Полный отказ от пакета `regexp` в пользу прямой проверки первого байта строки (`name[0] >= 'A'`).
- Это позволило устранить конкуренцию за мьютекс внутри движка регулярных выражений.

**Вывод:** Для простых проверок в высоконагруженных системах следует избегать регулярных выражений. Прямая работа с байтами и строками дает максимальный прирост RPS и снижает задержки.

### Команда запуска теста
```bash
  hey -n 20000 -c 50 -m POST -T "application/json" -D data.json http://localhost:8082/process
```

### 🛠 Как запустить локально

1. **Генерация тестовых данных**:
   ```bash
   go run gen.go
   ```
2. **Запуск сервера**:
    ```bash
   go run main.go api.go
   ```
3. Сбор профиля (CPU) во время нагрузки:go tool pprof http://localhost:8082/debug/pprof/profile?seconds=10
